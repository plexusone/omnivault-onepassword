## [v0.2.0] - 2026-03-14

### Highlights

- Module migrated to new GitHub organization: `github.com/plexusone/omnivault-onepassword`
- Updated for 1Password SDK v0.4.0 API breaking changes

### Breaking

- Module path changed from `github.com/agentplexus/omnivault-onepassword` to `github.com/plexusone/omnivault-onepassword` ([`84c7f51`](https://github.com/plexusone/omnivault-onepassword/commit/84c7f51))

### Upgrade Guide

- Update import paths from `github.com/agentplexus/omnivault-onepassword` to `github.com/plexusone/omnivault-onepassword`
- Run `go get github.com/plexusone/omnivault-onepassword@v0.2.0` to fetch new module

### Changed

- SDK API calls updated from field access to method calls (`client.Items` to `client.Items()`) ([`84c7f51`](https://github.com/plexusone/omnivault-onepassword/commit/84c7f51))
- List operations changed from iterator pattern to direct slice returns ([`84c7f51`](https://github.com/plexusone/omnivault-onepassword/commit/84c7f51))

### Dependencies

## [v0.1.0] - 2026-02-10

### Highlights

- Initial release of OmniVault provider for 1Password

### Added

- OmniVault `vault.Vault` interface implementation for 1Password
- Full CRUD operations (Get, Set, Delete, Exists, List)
- Batch operations support (GetBatch, SetBatch, DeleteBatch)
- Multi-field item support with automatic field type inference
- TOTP code generation and extraction
- Flexible path formats including native `op://` references
- Vault and item name-to-ID resolution with caching
- Tag support with key:value formatting
- Comprehensive error mapping to OmniVault error types
